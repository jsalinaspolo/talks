Event Sourcing

With event sourcing, instead of storing the “current” state of the entities that are used in our system, we store a sequence of events that relate to these entities.

Events are immutable

Log

A Log is immutable, ordered, append-only and persistent data structure. It’s a very simple structure, but it is very powerful.

Logs have a specific purpose: they record what happened, when and where.

Kafka

Kafka is a distributed streaming platform

Order is guaranteed within a partition in a topic.

Each partition is a log.
