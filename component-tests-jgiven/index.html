Cucumber


Feature: Eating too many cucumbers may not be good for you

  Scenario: Eating a few is no problem
    Given Alice is hungry
    When she eats 3 cucumbers
    Then she will be full

Good

  • Written in English
  • Easy to define

Bad

  • Hard to maintain
  • Code is secondary
  • No interaction between Business and Development

Spock - BDD


def "Eating a few is no problem"(){
       given: "Alice is hungry"
           ...
       when: "she eats 3 cucumbers"
           ...
       then: "she will be full"
           ...
   }

Good

  • Groovy
  • Code and Specification comes together
  • Written by "developers"

Bad

  • Hard to re-use
  • DSL are hard to build

JGiven


    @Test public void
    eating_a_few_is_no_problem() {
      given().$_is_hungry("Alice")
      when().she_eats_$_cucumbers(3)
      then().she_will_be_full()
    }

Good

  • Code is first class citizen - Specification is the result
  • Written by developers
  • Communication between business and developers
  • Easy to maintain

Bad

  • Sometimes can be a bit verbose
  • Use annotations for sharing state between stages
  • Another library
